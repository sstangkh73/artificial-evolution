## 🔴 ใหม่-ร้ายแรง

### R2-1. "lure/per-type learning fails" — เป็นสมบัติของ **gate เฉพาะตัวเรา** ไม่ใช่ของ "การเรียนรู้ต่อชนิด" ทั่วไป (จุดที่ reviewer เก่งจะยิงแรงสุด)
ต้องแยก 2 ระดับให้ขาด:
- **discrimination = 0 → แข็งระดับพิสูจน์ทาง analytic:** ค่าใด ๆ ที่ key ด้วย "ชนิด" ให้ค่าเดียวต่อชนิด → การตัดสินใจเท่ากันทุกอายุ **เสมอ** ไม่ว่ากฎอัปเดตอะไร → discrimination เป็น 0 โดยโครงสร้าง ✅ อ้างได้เต็มปาก
- **แต่ lured% (63%), blend, "แย่กว่าพิษคงที่" = ขึ้นกับกฎเรียนรู้เฉพาะของเรา** (`_food_worth_eating`: ชิมของใหม่ครั้งเดียว→แช่แข็งค่า→เกณฑ์แข็ง pickiness×best + EMA) reviewer จะพูดว่า **"นี่ไม่ใช่ 'per-type learning แพ้' แต่เป็น 'gate ที่แช่แข็งค่าจากคำแรกและไม่ sample ซ้ำ แพ้'** — ε-greedy / softmax / optimistic-init / learner ที่ sample ต่อ อาจฟื้นตัวหรือ lure หายไป"
**แก้:** (ก) จำกัดคำเคลมให้ตรง — "ตัวประมาณค่าที่ key ด้วยชนิดใด ๆ แยกสภาพไม่ได้ (analytic); *ขนาด* ของ lure สาธิตด้วย gate ของเราและจะแปรตาม learner" **หรือ** (ข) **ทดสอบ learner มาตรฐาน ≥1 ตัว** (softmax/ε-greedy) โชว์ว่า lure ไม่ใช่ artifact ของการแช่แข็ง — อันนี้จะทำให้ผลแข็งขึ้นมาก

## 🟠 ใหม่-กลาง

### R2-3. "so what" ของ lure ยังไม่พิสูจน์ — เป็น artifact ของโหมดอิ่ม
ในโหมดอิ่ม agent **ไม่หิว** → การถูกล่อไปกินพิษบ้าง **ยังไม่มีต้นทุน fitness ที่แสดงให้เห็น** (มันไม่จำเป็นต้องกินผลไม้เลยก็ได้) gap-vs-optimal (76%) เป็น "ความผิดพลาดของ preference" ที่ยังไม่โยงกับการรอด/การถูกคัด → คำเคลม "อันตรายกว่าพิษคงที่" ตอนนี้เป็นเรื่อง **ค่าที่รับรู้ ไม่ใช่อันตรายที่พิสูจน์** ว่า lure "สำคัญ" ในระบบเต็มยัง untested
**แก้:** เปิดเผยตรง ๆ ว่า fitness cost ของ lure ยังไม่ทดสอบ (เป็น future work); อย่าให้ "อันตรายกว่า" ฟังเหมือนพิสูจน์การตาย

### R2-4. Aging ไม่เคยถูกพิสูจน์ว่า "ช่วย" ปัญหา boom-bust ที่มันถูกสร้างมาแก้
desync (CV 0→1.56) ใช้ยีน **สุ่ม uniform (ไม่ได้วิวัฒน์)** และ **ไม่มีการทดลองว่าเปิด aging ประชากรรอดดีกว่าปิด** (ทั้งคู่สูญพันธุ์ในผลที่มี) → คำเคลมจูงใจ "aging ช่วย boom-bust" เป็นการ **สมมติ ไม่ได้ทดสอบ**
**แก้:** ระบุว่ายังไม่ทดสอบ; หรือรัน aging-on vs off เทียบการอยู่รอด (แต่ติดคอขวด foraging เดิม)

### R2-5. "Aging validate เทียบงานวิจัย 4/4" ขายเกิน self-consistency
exponent 0.25 เป็น **knob** → "validated against biology" สื่อว่ายืนยันอิสระ (ทั้งที่เป็น self-consistent) complete report เขียน caveat ในข้อจำกัดแล้ว **แต่ยังพาดหัว "4/4 validated" ใน abstract/results** → ควรลดเป็น "สร้างกฎออกมาโดยโครงสร้าง / sanity-check ว่าโมเดลทำตามที่ออกแบบ"

### R2-6. ปน 2 regime อาหารโดยไม่บอกชัด (§5.2 พิษคงที่ vs §5.3 พิษตามอายุ)
กราฟ avoidance (fig4/5) ใช้ **พิษคงที่**; lure (fig6/7) ใช้ **พิษตามอายุ** — complete report ไม่บอกชัดว่าเป็นอาหาร**คนละแบบ** ผู้อ่านละเอียดจะสับสนว่าเป็นผลไม้ตัวเดียวกันไหม
**แก้:** ระบุให้ชัดว่า §5.2 = baseline พิษคงที่ (การเลี่ยงทำงาน), §5.3 = พิษตามอายุ (การเลี่ยงพัง)

### R2-7. เรื่องเล่าไม่ตรงกัน 2 เอกสาร: complete report นำด้วย aging / ALife abstract ตัด aging ทิ้ง
คนอ่านทั้งคู่จะงงว่า thesis กลางคืออะไร (aging หรือ toxin?) — ควรให้ complete report **กรอบ aging เป็น "โครงสร้างพื้นฐานของแพลตฟอร์ม" ไม่ใช่ผลเด่น** แล้วชู toxin/lure เป็นผลหลัก ให้สอดคล้อง abstract
